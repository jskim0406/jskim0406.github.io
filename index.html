Interlacing of eigen-value

January 30, 2021 Math_python

행렬 A가 변하면, 그 고유값은 어떻게 될까?

단도 직입적으로, 이에 대한 답을 엿볼 수 있는 개념이 **’고유값의 interlacing’**이다.

고유값의 interlacing

정의

[‘고유값의 interlacing’ 정의]

$S + uu^{T}$의 각 고유값 $z_{i}$는 $\lambda_{i}$이상이고 $\lambda_{i-1}$이하이다. 따라서, $\lambda$들과 $z$들이 서로 엇갈린다(interlaced). 각각의 $z_{2}, …, z_{n}$은 두 개의 $\lambda$ 사이에 있다. ($S$ = 대칭행렬, $uu^{T}$ = rank-1 행렬, $\lambda$ = $S$의 고유값)

$$z_{1} \geq \lambda_{1} \geq z_{2} \geq \lambda_{2} \geq … \geq z_{n} \geq \lambda_{n}$$

풀어보자면, 대칭행렬 S에 ‘특정한 변화’가 주어지면, 고유값은 변화 이전과 이후의 것들이 서로 엇갈리게 된다는 것이다.

이번 포스팅은 ‘고유값의 interlacing’을 엄밀한 수학적 정의로 풀어내기 보다, 보다 직관적으로 이해하는 데 목표를 두고자 한다.

rank-1 행렬의 특징

이 주어진 ‘변화’에 조금 더 주목할 필요가 있다. 몇가지 더 살펴보자.

1. $uu^{T}$는 왜 rank-1 행렬인가?

S에 주어진 ‘특정한 변화($uu^{T}$)’는 rank-1 행렬로 주어졌고, 이는 대칭행렬이 된다. 왜 rank-1 행렬이 대칭행렬이 될까? 이는 간단한 예로 바로 확인할 수 있다.

\begin{aligned}u=\begin{bmatrix}
1 \
0
\end{bmatrix}일 때, uu^{T}=\begin{bmatrix}
1 \
0
\end{bmatrix}\begin{bmatrix}
1 & 0
\end{bmatrix}=\begin{bmatrix}
1\cdot 1 & 0\cdot 1 \
1\cdot 0 & 0\cdot 0
\end{bmatrix}
=\begin{bmatrix}
1 & 0 \
0 & 0
\end{bmatrix}\end{aligned}

이와 같이, $uu^{T}$는 랭크가 1인 행렬이 된다.

2. “왜 ‘rank-1 행렬’이 더해졌는 가?

처음 공부하며 가장 의아했던 부분이다. 생각해보면 간단하다. 행렬의 각 요소에 특정한 값을 가감해주려면, ‘변화’는 당연하게도 같은 shape의 행렬이 되어야 하고, 가장 간단하고 심플한 rank-1 행렬이 가감되는 상황을 이야기하고 있다.

3. rank-1 행렬 = 대칭행렬 = 양의 준정부호

rank-1 행렬은 대칭행렬이다. 대칭행렬은 양의 준정부호를 갖는다. 이 부분에서 가장 중요한 점을 볼 수 있다. 행렬 $S$에 주어진 변화가 바로 ‘양의 준정부호’**라는 점이다. 즉, **기존의 행렬에 0 혹은 + 의 변화를 준 것이다. 그렇다면 기존 행렬의 고유값은 늘어날까? 줄어들까?

‘양의 준정부호’ 변화의 의미 = function 값의 scaling

주어진 변화가 0 혹은 + 이다. 하지만, 행렬에 주어진 변화가 0 혹은 + 라는 점과 행렬이 부호를 갖는 다는 게 조금 생소할 수 있다. 이해를 돕기 위해, 먼저 ‘함수’의 관점에서 이를 살펴보자.

결국, 행렬이란 선형인 함수와 같기 때문에, 이러한 이해가 도움이 될 수 있을 것이다.

$$f_{1}=x^{2}+1, f_{2}=x^{2}$$
$$f_{3}=f_{1}+f_{2}=2x^{2}+1$$

위와 같은 함수를 생각해보자. $f_{2}$는 입력 값($x$)이 무엇이든, 출력 값은 항상 0 혹은 양수이다. 즉, 이 함수의 ‘부호’는 양의 준정부호이다.

$f_{3}$이라는 함수는 $f_{1}$에 ‘양의 준정부호’ $f_{2}$를 더해준 것이다.

즉, 인터레이싱에서 다루는 변화를 간단한 함수로 재현한 것과 같다. 과연, $f_{3}$는 $f_{1}$에 비해 어떻게 변해있을 까? 그래프를 그려 확인해보자

5 4 9

위의 그래프를 보면, $f_{2}$를 더해주는 것이 어떤 의미를 갖는 지 새삼 다시 확인할 수 있다.

$x=2$ 일 때의 값을 보자. 양의 준정부호인 $f_{2}$를 더해줌으로써, function값이 더 늘어났음을 볼 수 있다.
$f_{3}$그래프(빨간색)을 보면, $f_{1}$값(초록색)에 비해 커졌음을 알 수 있다.

즉, $f_{2}$(양의 준정부호 값)를 더해줌으로써 $f_{1}$값이 positive하게 scaling 된 것이다.

이것을 통해 양의 준정부호 변화가 주어졌을 때의 효과를 매우 단편적이지만 엿볼 수 있다.

행렬 = linear mapping = function

행렬이란 linear mapping의 과정이다. 그리고 이는 function이다.
linear mapping 이란 다음과 같이 이해할 수 있다.

$$\forall x,y \in V, \forall a,b \in \mathbb{R}, f : V \rightarrow W$$

$$f(ax+by) = af(x) + bf(y)$$

위를 만족할 경우, $f$ 는 linear mapping이고, 이는 자연스럽게 function의 역할을 하게 된다.
그리고 이 funciton $f$는 벡터공간 V와 W사이의 transformation이고, 이는 행렬의 역할과 동일하다. 즉, 행렬은 linear mapping의 역할을 하는 일종의 function인 것이다.

‘양의 준정부호’ 변화의 의미 = 행렬 mapping 효과의 scaling

함수에서 살펴본 변화가 행렬에도 동일하게 적용된다.

함수에서는 function값이 늘어났다면, 행렬에선 mapping의 scaler인 ‘고유값’이 변하게 되는 것이다.

이를 데카르트 좌표계에서 직접 확인해보자

아래와 같이, 가장 간단한 행렬 A를 놓고 살펴보자.

$A = \left[ \begin{array}{cc} 2&0 \ 0&2 \end{array} \right]$
(identity matrix, 대칭행렬)

A에 양의 준정부호 변화가 더해지면, A의 고유값은 어떻게 될까?

먼저, 변화가 더해지기 전의 행렬 A를 살펴보자.

x : 
[[1]
 [1]]
A : 
[[2 0]
 [0 2]]
y = Ax : 
[[2]
 [2]]
eigen value of A : 
[2.0, 2.0]
eigen vector of A : 
[[1. 0.]
 [0. 1.]]

만약, A에 다음과 같은 변화가 생긴다면?

$$A + uu^{t} = \tilde{A}$$

$$A+uu^{T}=\begin{bmatrix} 2 & 0 \ 0 & 2 \end{bmatrix}+\begin{bmatrix} 1 & 0 \ 0 & 0 \end{bmatrix}=\begin{bmatrix} 3 & 0 \ 0 & 2 \end{bmatrix} = \tilde{A}$$

기존의 행렬 $A$에 랭크-1행렬을 추가해주었다. 이로써 새로운 행렬 $\tilde{A}$를 얻었다. 랭크-1행렬은 대칭행렬이다. 대칭행렬은 positive semidefinite이다. 즉, 랭크-1행렬은 pos-semidef이고, 고유값은 모두 0보다 크거나 같다.

기존의 행렬에 0 혹은 +의 변화를 주었다. 그렇다면, 기존 행렬의 고유값은 늘어났을까 줄어들었을까?

직관적으로 생각해보면, 새로운 행렬 $\tilde{A}$의 고유값이 기존보다 늘어났을 것 같다.
왜냐하면 기존 행렬에 양부호의 값을 더해주었으니, scaler인 고유값도 더 늘어났을 것 같기 때문이다.
실제로 그런지 살펴보자.

x : 
[[1]
 [1]]
A_tilde : 
[[3 0]
 [0 2]]
y_tilde : 
[[3]
 [2]]
eigen value of A_tilde : 
[3.0, 2.0]
eigen vector of A_tildes : 
[[1. 0.]
 [0. 1.]]

위와 같이, $\tilde{A}$를 곱해주면 그 결과는 달라진다. 달라지는 이유는 $\lambda_{1}$에서 찾을 수 있다.

직접 비교하기 위해 두 그래프를 함께 보자

고유값 중 가장 큰 고유값인 $\lambda_{1}$의 값이 2배가 되면서 대응하는 첫 고유벡터의 방향으로 mapping 결과가 기울어진 것을 볼 수 있다.

이를 요약하면 아래와 같다.

$$\tilde{\lambda_{1}} \geq \lambda_{1} \geq \tilde{\lambda_{2}} \geq \lambda_{2}$$

Interlacing

그리고 이를 위와 같은 2x2 행렬이 아닌, nxn행렬로 확장해 일반화한 것이 바로 ‘interlacing’이다.

마지막으로 처음에 봤던 정의를 다시 보아보자.

[‘고유값의 interlacing’ 정의]

$S + uu^{T}$의 각 고유값 $z_{i}$는 $\lambda_{i}$이상이고 $\lambda_{i-1}$이하이다. 따라서, $\lambda$들과 $z$들이 서로 엇갈린다(interlaced). 각각의 $z_{2}, …, z_{n}$은 두 개의 $\lambda$ 사이에 있다. ($S$ = 대칭행렬, $uu^{T}$ = rank-1 행렬, $\lambda$ = $S$의 고유값)

$$z_{1} \geq \lambda_{1} \geq z_{2} \geq \lambda_{2} \geq … \geq z_{n} \geq \lambda_{n}$$

$z_{i} = \tilde{\lambda_{i}}$로 치환하면, 우리가 보았던 결과와 같게 된다.

아스키 코드, 유니코드, UTF-8 그리고 Python

[Reference]

  • [블로그] : URL
  • [서적] “파이선 알고리즘 인터뷰(박상길)”

기본적인 배경지식

  1. 컴퓨터의 기본 저장 단위 : Byte

  2. 1 byte = 8 bit = $2^{8}$(256)개의 값 저장 가능

  3. 문자 / 기호 집합을 컴퓨터에서 표현하려면 부호(code)로 바꿔야 하는 데, 이를 ‘encoding’이라 한다.

    • ex) 모스부호

  4. 이러한 encoding 방식에 ASCII, UNICODE, UTF-8 등이 있다.

1) ASCII (American Standard Code for Information Interchange)

  • (8-1 = 7)비트로 문자/부호 표현
  • 8비트 중 1비트는 통신에러 검출을 위한 ‘parity bit’로 사용되어, 실질적인 정보는 7비트에 담아 표현한다.
  • 즉, 1byte에는 $2^{8}$개가 아닌 $2^{7}$개(128)의 정보를 담을 수 있다.

[출처] : URL

한계 : $2^{7}$로는 영어 외 다른 언어 문자를 표현하기에 어려움

  • 전세계 언어 문자를 모두 담을 수 있는 통일된 인코딩 체계로 만들어진 것이 ‘UNICODE’

2) UNICODE

  • 한글 : 자음(21개) + 모음(19개) 조합만 해도 128개를 초과(단순 조합 수 계산 시, $_{40}C_2$ 개의 조합이 가능)

  • 중국어 : 한자의 갯수는 128개를 초과

    • 따라서, ASCII코드(1byte 단위)로는 한글, 중국어 문자를 담아낼 수 없다

  • UNICODE : 1byte(8bit) -> 2 ~ 4byte(16~32bit)로 저장공간을 넓혀 문자를 할당한다.

한계 : 유니코드는 공간이 ‘고정길이’를 갖는 ‘고정길이 인코딩’이다. 따라서, 공간 효율성이 낮다.

​ ex) 영문은 ASCII 문자 체계에 해당해, 예전과 같이 1 byte에 담을 수 있지만 UNICODE에 따르면 무조건 2byte를 할당해야 하는 것이다.

고정길이 인코딩에서 벗어나 가변길이 인코딩으로 공간 효율성을 높인 인코딩 방식이 **’UTF-8’**이다.

3) UTF-8

  • 유니코드 값에 따라 가변적으로 할당할 Byte 크기를 결정해 인코딩하는 방식 (8bit 단위로 결정하기 때문에 ‘UTF-8’이다. UTF-16도 있음)

ex)

  • ‘A’의 유니코드 값 <= 1byte
    따라서, 1 byte로 인코딩

  • 특수문자의 유니코드 값 ( 1byte <= <= 2byte )
    따라서, 2 byte로 인코딩

  • ‘한’의 유니코드 값 ( 2byte <= <= 3byte )
    따라서, 3 byte로 인코딩
